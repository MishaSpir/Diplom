
# Как настроить проект на работу с новым контроллером
Например, если раньше работал с 407, а тепеь у тебя Дискавери с 303 контроллерром.
0. Предполагается , что уже есть папка с проектом с подпапками и файлами, если нет то лучше стырить ее у того, у кого уже есть, если такого нет, то придется скачивать все из инета.
	==**project**==
		==build== ------------эта папка формируется сама, посе того как первый раз подключаются файлы(компилируется проект)
		==**cmake**==
		==**doc**== 
		**==flash_n_debug==**--
		==**libs**==--------------тут лежитт сама libopencm3
		==**src**==--------------тут лежат наши исходники, то есть файлы .cpp , в который мы пишем программу.
		==**.git**== -------------необязательно, нужно если использовать систему контроля версий ГИТ, формируется ГИТом
		==**.vscode**== --------тут лежат 
		CMakeLists.txt--обязательно, именно в этом файле прописывается, какой контроллер мы будем использовать
		тут лежит launch.json нужный для запуска и отладки, там тоже надо менять файл svd для нового контроллера, там может быть и несколько конфигураций для сразу нескольких контроллеров и при отладки нуную надо будет выбрать в VSCode.
		README.md----также формируется ГИТом
		git.ignore-------также формируется ГИТом
	
	Также предполагается, что установлены все libopencm,arm-none-eabi и прочее...

1. Итак для работы с новым контроллером нужно вначале исправить файл CmakeLists.txt. Например для перехода на STM303:
```
#================================================================================
cmake_minimum_required(VERSION 3.20)
set(CMAKE_TOOLCHAIN_FILE cmake/arm-none-eabi-gcc.cmake)


#================================================================================
project(es LANGUAGES CXX C ASM)
set(DEVICE stm32f303vc)


#================================================================================

add_subdirectory(src)

add_subdirectory(libs)

add_subdirectory(flash_n_debug)

add_subdirectory(doc)
```
2. Также для отладки (ЭТО НЕОБЯЗАТЕЛЬНО, ПРОЕКТ БУДЕТ СОБИРАТЬСЯ И ПРОШИВАТЬСЯ БЕЗ ЭТОГО) необходимо изменить launch.json. Нужно добавить новую конфигурацию для 303 контроллера. Для этого вначале скачиваем файл stm32f303.svd и кладем его в директорию flash_n_debug. Редактируем launch.json:

## Если копируешь папку с проектом:
- удали папку build в новой папке проекта 
- после того, как открыл проект в VSCode выбери unspecifeid (неуказанный) или, если никаких выпадающих списков нет, то:
	- `Ctrl+Shift+P` -> `CMake: Select a Kit` и выбери unspecifeid
- введи в CmakeLists.txt назввание проекта

# UART 

асинхронный режим работы - приемник понимает данные только по временным интервалам, пожтому важно выставлять boudrate = битрейт =бит в секунду.
в самом простом вариант мы работаем с 3-мя регистрами
DATA REGISTER - в него записываем данные 

В микроконтроллерах STM32 передача данных по UART может осуществляться в блокирующем (polling) и неблокирующем (interrupt или DMA) режимах.

***Блокирующий (Polling) режим:***
В этом режиме передача данных происходит путем последовательной отправки каждого байта и ожидания завершения передачи, прежде чем перейти к следующему. Для этого используется функция HAL_UART_Transmit, которая блокирует выполнение кода до завершения передачи:
```
HAL_UART_Transmit(&huart1, (uint8_t*) "Hello, STM32!\r\n", 15, 100);
```
Этот режим подходит для небольших объемов данных, но не эффективен при передаче больших массивов, так как процессор большую часть времени простаивает в ожидании. Пользоваться крайне нежелательно, так как эта функция будет тормозить код также как и DELAY


***Неблокирующий режим (с использованием прерываний):***
Режим прерываний позволяет передавать данные, не блокируя выполнение основного кода. Когда начинается передача данных, функция HAL_UART_Transmit_IT запускает процесс и возвращается. Когда передача каждого байта завершается, генерируется прерывание, и обработчик прерывания (`HAL_UART_TxCpltCallback`) отправляет следующий байт.
```
HAL_UART_Transmit_IT(&huart2, (uint8_t*) "Hello, STM32!\r\n", 16);
```
