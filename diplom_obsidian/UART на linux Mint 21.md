Была проблема, с  COM портами: когда вставлял в порт USB-UART он на секунду определялся, а потом пропадал. Оказалось, его перехватывала какая-то программа к порту невозможно было подключиться. Программа BRLTTY (для Braille-дисплеев) перехватывала вашу Arduino с чипом CH340.

``/dev/ttyUSB0`` открывался и сразу закрывался.

Я тестировал на Arduino nano. Отключил плату , выполнил в терминале 

```bash
sudo udevadm monitor --environment --udev
```
Подключился к нано и увидел вывод логов
В 90% случаев порт "воруют" две стандартные программы для Linux, которые нужны для людей с ограниченными возможностями. Они ошибочно принимают вашу Arduino за специальное устройство ввода.
**1. Сервис BRLTTY (для Braille-дисплеев)**
В логе четко видно:
1. **Сначала порт успешно создается** - появляется `/dev/ttyUSB0`
2. **BRLTTY перехватывает устройство** - строка `BRLTTY_BRAILLE_DRIVER=bm`
3. **Сразу после этого порт удаляется** - `ACTION=remove` для `/dev/ttyUSB0`

**Вариант  - удалить файл (радикально):**
```bash
sudo rm /lib/udev/rules.d/90-brltty.rules
```
 **Шаг 3: Перезагрузите udev-правила**
```bash
sudo udevadm control --reload-rules
sudo udevadm trigger
```
